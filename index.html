5 formulas
VLOOKUP Lookup
Look up a value and return from another columnWert suchen und aus anderer Spalte zurückgeben
+
Syntax
=VLOOKUP(A2, $B:$D, 2, FALSE) → always use FALSE for exact match ⚠ Can only look RIGHT — use XLOOKUP or INDEX/MATCH for left ⚠ col_index breaks if columns are inserted
What it doesWas es macht
Searches for a value in the first column and returns a value from a specified column in the same row.Sucht einen Wert in der ersten Spalte und gibt einen Wert aus einer anderen Spalte zurück.
Common mistakesHäufige Fehler
1. Using TRUE (use FALSE)
2. Not locking: $B$2:$D$1001. TRUE verwenden (FALSE verwenden)
2. Nicht sperren: $B$2:$D$100
XLOOKUP Lookup
Modern replacement for VLOOKUP — cleaner & more powerfulModerner SVERWEIS-Ersatz
+
Syntax
=XLOOKUP(lookup, lookup_range, return_range, [if_not_found]) =XLOOKUP(A2, B:B, C:C, "Not found") ✓ Can look left, right, up or down ✓ Handles missing values natively — no IFERROR needed ✓ Does not break when columns are inserted
vs VLOOKUPvs SVERWEIS
No col_index needed — point directly to the return range. Use XLOOKUP over VLOOKUP whenever on Excel 365.Keine col_index-Nummer nötig. Bei Excel 365 immer XLOOKUP statt SVERWEIS verwenden.
INDEX + MATCH Lookup
Look in any direction — left or rightIn jede Richtung suchen
+
Syntax
=INDEX(return_range, MATCH(lookup_value, lookup_range, 0)) =INDEX(C:C, MATCH(A2, B:B, 0)) → Find A2 in column B, return from column C → Can look LEFT — VLOOKUP cannot
INDEX
Returns value at a given position. "Give me the Nth item."Gibt Wert an einer bestimmten Position zurück.
MATCH
Finds the row number. Together: MATCH finds position, INDEX retrieves value.Findet die Zeilennummer. Zusammen: VERGLEICH findet Position, INDEX holt Wert.
INDIRECT Lookup
Reference a cell dynamically using a text stringZelle dynamisch über Text referenzieren
+
Syntax
=INDIRECT("A"&B1) → if B1=5, returns value of A5 =INDIRECT("'"&A1&"'!B2") → reference another sheet dynamically =SUM(INDIRECT("A1:A"&C1)) → dynamic range in SUM
What it doesWas es macht
Turns a text string into a real cell reference. Lets you build references dynamically — great for dashboards.Wandelt Text in einen echten Zellbezug um. Ideal für dynamische Dashboards.
Common useHäufige Verwendung
Dropdown with sheet name → shows data from that sheet automatically.Dropdown mit Blattname → zeigt automatisch Daten dieses Blatts.
OFFSET Lookup
Return a range shifted by rows and columnsBereich um Zeilen und Spalten verschoben
+
Syntax
=OFFSET(reference, rows, cols, [height], [width]) =OFFSET(A1, 2, 1) → B3 (2 down, 1 right from A1) =SUM(OFFSET(A1,0,0,COUNTA(A:A),1)) → auto-expanding SUM
What it doesWas es macht
Returns a reference shifted from a starting point. Can return a dynamic range — great for charts that grow automatically.Gibt einen verschobenen Bezug zurück. Kann einen dynamischen Bereich zurückgeben — ideal für automatisch wachsende Diagramme.
vs INDIRECT
INDIRECT builds from text. OFFSET shifts from a cell. Use OFFSET when you know how many rows/cols to move.INDIREKT baut aus Text. BEREICH.VERSCHIEBEN verschiebt von einer Zelle. Verwenden, wenn Zeilen-/Spaltenanzahl bekannt.

AND / OR / NOT Logical
Test multiple conditions at onceMehrere Bedingungen gleichzeitig testen
+
Syntax
=AND(cond1, cond2) → TRUE only if ALL are true =OR(cond1, cond2) → TRUE if ANY is true =NOT(condition) → reverses TRUE/FALSE =IF(AND(A1>0, B1>0), "Both positive", "No")
What it doesWas es macht
Almost always used inside IF. AND = all must match. OR = at least one must match.Fast immer in WENN verwendet. UND = alle müssen passen. ODER = mindestens eine muss passen.
IFERROR / IFNA Logical
Handle errors gracefullyFehler elegant behandeln
+
Syntax
=IFERROR(formula, value_if_error) =IFERROR(VLOOKUP(A1,B:C,2,0), "Not found") =IFERROR(A1/B1, 0) → return 0 if division by zero =IFNA(formula, "") → only catches #N/A errors
What it doesWas es macht
Wraps any formula and returns a friendly value instead of an error. Essential for clean spreadsheets.Umhüllt jede Formel und gibt statt eines Fehlers einen freundlichen Wert zurück.
IFNA vs IFERROR
Use IFNA for lookups — only catches "not found", so real errors like #REF! still surface.WENNNA für Lookups — fängt nur "nicht gefunden" ab, echte Fehler bleiben sichtbar.
